## 1. Terms

### A

**Arbitrage**
Trading strategy that profits from price differences of the same asset across different markets or instruments.

**API (Application Programming Interface)**
Set of protocols and tools for building software applications, specifying how software components interact.

### B

**Bid**
The price buyers are willing to pay for an asset. The highest price a buyer will accept.

**Bridge**
Protocol that enables transferring assets between different blockchains.

**Bull Market**
Market condition where prices are rising or expected to rise.

**BPS (Basis Points)**
Unit of measure equal to 1/100th of a percentage point. Used for spreads, fees, and profits.

### C

**CEX (Centralized Exchange)**
Cryptocurrency exchange operated by a central authority (e.g., Binance, Coinbase).

**Circuit Breaker**
Automatic mechanism that halts trading when predefined conditions are met (e.g., 3 consecutive failures).

**Confidence Score**
Metric (0-1) indicating how likely an opportunity is to be profitable after execution.

**Connector**
Software component that interfaces with a specific exchange or data source.

**Cross-Chain Arbitrage**
Arbitrage between the same asset on different blockchains, requiring bridge transfers.

**CTP (Cancel-Trade-Place)**
Execution strategy where an existing order is canceled and replaced at a better price.

### D

**DEX (Decentralized Exchange)**
Cryptocurrency exchange operating on blockchain without central authority (e.g., Uniswap).

**Discovery**
Process of finding available trading pairs, assets, and liquidity sources.

**Dry Run**
Simulation of a trade without actual execution, used for validation.

### E

**EOA (Externally Owned Account)**
Blockchain account controlled by a private key, as opposed to a smart contract.

**Execution**
Process of submitting and filling trades on exchanges.

### F

**Factory Contract**
Smart contract that creates new pools or tokens on DEXes.

**Fee Tier**
Different fee structures on DEX pools based on volatility expectations.

### G

**Gas**
Fee paid to execute transactions on blockchain networks.

**GTE (Greater Than or Equal)**
Comparison operator used in queries to filter minimum thresholds.

**Guardrail**
Safety mechanism that prevents dangerous trades based on predefined rules.

### H

**Hash**
Cryptographic fingerprint of data, used for verification and integrity.

**HFT (High-Frequency Trading)**
Trading strategy using high-speed systems to execute many trades in milliseconds.

### I

**Idempotency**
Property where multiple identical requests have the same effect as a single request.

**Incentive**
Reward mechanism (e.g., liquidity mining, airdrops) that affects market pricing.

### J

**JWT (JSON Web Token)**
Compact, URL-safe token used for authentication and information exchange.

### K

**Kill Switch**
Emergency stop mechanism that halts all automated trading.

### L

**L2 (Layer 2)**
Scaling solution built on top of base blockchain (e.g., Arbitrum, Optimism).

**Liquidity**
Availability of an asset to buy or sell without significant price impact.

**LP (Liquidity Provider)**
Entity that deposits assets into a pool to enable trading.

**LTV (Loan-to-Value)**
Ratio of loan amount to collateral value, used in lending protocols.

### M

**MEV (Maximal Extractable Value)**
Profit extracted by reordering, inserting, or censoring transactions in a block.

**Mempool**
Holding area for unconfirmed transactions before inclusion in a block.

### N

**Nonce**
Number used once in cryptographic communications, prevents replay attacks.

### O

**Order Book**
List of buy and sell orders for an asset, organized by price.

**Opportunity**
Detected price difference that can be exploited for profit.

### P

**Pair**
Trading pair representing exchange rate between two assets (e.g., BTC/USDT).

**Perp (Perpetual Futures)**
Derivative contract with no expiration date, used for speculation or hedging.

**Pool**
Smart contract holding liquidity for trading on DEXes.

**Price Impact**
Change in asset price caused by a trade, larger trades have higher impact.

**Protocol**
Set of rules governing blockchain operations.

**psql (PostgreSQL)**
Open-source relational database system.

### Q

**Quote**
Price at which an asset can be bought or sold.

### R

**RPC (Remote Procedure Call)**
Protocol for calling functions on remote systems (e.g., blockchain nodes).

**Risk Score**
Numerical assessment (0-100) of opportunity safety.

**RLS (Row-Level Security)**
PostgreSQL feature restricting row access based on user permissions.

**Rollback**
Reverting to a previous state or version.

### S

**Slippage**
Difference between expected and actual execution price.

**Smart Contract**
Self-executing code deployed on blockchain.

**Spread**
Difference between bid and ask prices.

**Stablecoin**
Cryptocurrency pegged to stable asset (e.g., USDC to USD).

**Subgraph**
Indexed blockchain data queryable via GraphQL.

**Slippage**
Difference between expected and actual execution price.

### T

**Ticker**
Unique symbol identifying a trading pair (e.g., BTC).

**TPS (Transactions Per Second)**
Measure of blockchain throughput.

**TVL (Total Value Locked)**
Total assets deposited in a protocol.

### U

**USDⓈ-Margined**
Futures contract settled in stablecoin rather than crypto asset.

### V

**VWAP (Volume-Weighted Average Price)**
Average price weighted by trading volume over time period.

### W

**WETH (Wrapped ETH)**
ERC-20 representation of ETH for use in smart contracts.

**WebSocket**
Protocol for full-duplex communication channels over TCP.

### Z

**Zero-Knowledge Proof**
Cryptographic method proving something is true without revealing information.

## 3. Technical Terms

### 3.1 Trading

**Arbitrage Detection**
Algorithm identifying price discrepancies across markets.

**Flash Loan**
Uncollateralized loan within single transaction.

**Limit Order**
Order to buy/sell at specific price or better.

**Market Order**
Order to buy/sell immediately at best available price.

**Order Book Depth**
Total buy/sell orders at each price level.

**Slippage Tolerance**
Maximum acceptable difference between expected and execution price.

### 3.2 Blockchain

**Gas Fee**
Transaction fee paid to validators.

**Smart Contract**
Code deployed on blockchain executing automatically.

**Token Standard**
Specification for token implementation (ERC-20, ERC-721, etc.).

**Wallet**
Software/hardware storing private keys.

### 3.3 Infrastructure

**Container**
Lightweight, standalone software package.

**Containerization**
Packaging software with dependencies.

**Orchestration**
Automated deployment and management of containers (Kubernetes).

**Service Mesh**
Infrastructure layer for service-to-service communication.
